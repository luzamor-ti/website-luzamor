Resumo do projeto

O site é construído com Next.js App Router, Sanity CMS e TypeScript. O padrão é simples: CMS → serviços → páginas → componentes → UI. As páginas são definidas em código, e os conteúdos dinâmicos sempre têm fallback para evitar quebra visual.

Arquitetura

CMS (Sanity) → Services → Pages → Components → UI
  • Dados são buscados apenas em services.
  • Componentes de seção não fazem fetch direto.
  • Textos dinâmicos precisam de fallback em `constants/textFallbacks.ts`.
  • Schema e tipos devem permanecer sincronizados.

Começar rápido

  1. Instalar dependências com npm install.
  2. Configurar .env.local.
  3. Rodar npm run dev.
  4. Abrir o Studio em /fundacao-cms.

Storybook continua disponível via npm run storybook.
